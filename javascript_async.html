Why is it required?
Reading from a file system, or getting things from an external API or retrival of data from memory sources other than the RAM. It has been noted that:

  • Processor Speeds and utilization is one of the main factors of computation speed
  • Processor determines the pace of the internal computation, and the operating system can also allocate the processor towards other tasks.

Synchronous: It allows for multiple things to run at the same time. When an action is called, the program continues to run. The program finishes exectution, and the action outputs are given to the program and the values are accordingly updated.
Situation: Program fetches 2 results from a network and combines results. So, in a nutshell, in synchronous, the program will partially execute, and then will wait for O/P of first reqest, and then wait for the outcome of a the second reqest.
Only then does the program continue it's execution. The output is proportional to sum of execution of both request processes, assuming that they have proper outputs.
The solution?
Threads.
Since multiple processors are present in a computer system, each processor corresponds to a given thread. Let each thread correspond to a given action. When program awaits the request results, then the actions can simultaneously start, instead of the seond starting after the first one ends.
Callback: In standard OOP, we can pass objects into functions. JS aloows us to pass functions inside function, in order to yield output.
  • A callback is a function that is executed after another function is compoletely executed.
  • JS allows functions as arguments, and treats functions as objects. Functions that are passed as arguments are called callback functions.
  • JS is event driven, and it will not wait for the outcome of each process.
